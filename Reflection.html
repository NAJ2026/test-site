Reflection

Before embarking on this project, I had no idea how complicated it is to get words and formatting to appear correctly on a screen. I though there would be one way to go about it, but markdown and html have proven to be very different. I find markdown easier to understand and use, probably because I remember doing some last year.
What I was able to accomplish was learning the html language and how that translates into visual elements. Html requires quite a bit more planning than I’m used to. It’s tedious. If I forget to include the backslash, the whole line gets messed up. I also couldn’t figure out italics for a few days.
The other thing that took me days was figuring out how GitHub worked in general. Online display is an entirely new format for me, so I had no idea what I was doing, even with the instructional videos and online resources. Then, I couldn’t get any of my work to actually show up on the site preview. It was frustrating as it did, word-for-word, what the videos suggested, but nothing would show. I had to walk away for a whole day before I came back to try again. GitHub is not very user friendly, in my opinion.
In the end, I think I got everything to show up. I did some major edits to the ReadMe feature, which I’m still not exactly sure what it does, but it was helpful in getting everything to display. I think my original html file had something wrong with it, which led to the other issues. However, I made something digital, through a lot of experimentation.
